  • 二进制转十进制：

  • 十进制转二进制：
    对整数和小数部分分开来转话，整数部分就不说啦。对小数部分，乘以2，取新数的整数部分，然后对新数的小数部分重复此操作，直到新数小数部分为零或到达精度上限
    比如：3.875（10） 整数3(10) =>11(2)

      小数部分    
  ==0.875 乘以2 =>1.750 取1==
  ==0.750 乘以2 =>1.500 取1==
  ==0.500 乘以2 =>1.000 取1==

    按先后顺序依次成为二进制数小数点后的第一位、第二位、第三位……
    所以3.875（10）=11.111（2）

    怎么理解十进制转二进制呢

    1. 实体蛋糕
      我们不妨将“1”看成一块蛋糕，0.5就是一半的蛋糕，等价于二进制的0.1

      相应的
      二进制 0.01 = 十进制0.25为四分之一的蛋糕
      二进制 0.001 = 十进制0.125为八分之一的蛋糕
      二进制 0.0001 = 十进制0.0625为十六分之一的蛋糕
      ……
      我们把正方形切一半，再对其中一半再切一半，再对这切出的图形再切一半……
      最后变成了这样

      每一位二进制小数对应的部分如图

      若该位为1则取这部分图形否则不取。

      现在我们照这样的方式对这块蛋糕切啊切，切分成无数块，第一块有一半大，以后每一块是前一块的二分之一。
      稍加思考可以发现我想要的任意大小蛋糕都可以通过取舍不同的块数来达到或无限近似达到。
      但我该选哪些蛋糕块呢？
      首先想到的是贪心，比如我要0.618块蛋糕，先挑最大的0.5，嗯合适，还要再选0.118，发现第二个0.25和第三个0.125过大了，于是挑第四个0.0625。看看剩下多少，0.0555，继续往下挑。

    2. 数学上简单粗暴的理解
      我们要转化一个整数位为零的十进制数，，对于它的二进制数，整数位也一定为零。
      此时我们把这个数翻倍，对于它的二进制数，根据二进制数的性质，其实是小数点右移一位。
      这样，它的二进制数小数点后第一位就变成整数位了，这个整数位要么是1要么是0（二进制数嘛）
      若是1，那这个十进制数整数必然也是1，若是0则必然是0，。因此通过看十进制的整数是1是0，我们就知道了小数点后第一个数是1还是0，再不断重复这个方法来知道后续的二进制数。
      比如：
      0.625（10）=0.101（2）
      ×2 1.250（10）=1.010（2）取1
      ×2 0.500（10）=0.100（2）取0 （每次去掉整数部分再×2，可以发现二进制数小数点每次右移一位）
      ×2 1.000（10）=1.000（2）取1
      结果就是0.101

      正因为有这样的转化问题，导致很多有限十进制小数对应到二进制就变成无限小数了。而下面说的float类型保存的只能是二进制，因此很多十进制小数只能近似储存，产生精度问题

      Float与double是怎么储存小数的

      有了二进制小数后，我们要把他储存在float或者double里面，它们是怎么储存二进制小数的呢。
      我们先来看看float。
      比如我要储存1101.011011（即十进制的13.421875）
      先写成科学计数法1.101011011 x 2^3

    .101011011 部分我们称为尾数
    3 称为指数

    Spoil alert : 没错，浮点数都是以指数形式被保存的
    Float中的格式为SEEE EEEE EMMM MMMM MMMM MMMM MMMM MMMM
    第一位S的符号位
    接下来一个字节（8bit）是指数位，这里注意，因为指数可正可负，所以IEEE规定，这里算出来的值减去127才是真正的指数。比如某个float变量2-9位（也就是E的位置）为1000 0001，十进制数为129，再减去127得2，所以它的指数为2。

    M的部分着对应尾数部分（因为是科学计数法，小数点前面一定只有一个数1，这个是默认的了），多余的位补零

    所以，13.421875在float保存为0100 0001 0101 0110 1100 0000 0000 0000
    拆开float 4个字节验证下~

    输出： 0 -64 86 65（由于是小端模式保存，故高位数字保存在高位地址）
    十进制结果对应二进制数字，验证完成。

    简谈精度问题

    对于float，因为尾数位只有23位，通过计算精度大致在6-7位这样子（整数+小数部分）所以听low的
    float毕竟小啊，如果需求没有那么大，就用float吧
