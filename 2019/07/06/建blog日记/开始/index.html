建blog日记

开始

  • 吐槽

    终于换了新家。。。花了一天多才搭好一个空房子。回过头来，很想骂那个搭在廉价服务器上的wordpress。那个编辑界面真的是，文章由一行行段落组成，段落里插入图片极其麻烦，并且排版十分醉人且你毫无办法（写个目录还要单开一段，间隔超大超不顺眼）。当然，更关键是服务器速度，30+ RMB per year 的服务器真是不敢恭维，点“编辑文章”疯狂刷新+等上一年才慢悠悠出来界面，写文章的热情早已被稀释殆尽。

    还是GitHub + hexo大法好，免费+快速编辑（因为在本地）整个网站系统相对轻便，整个流程十分简洁。重要的是，Markdown！找的出比markdown更灵活的编辑方式，我切木薯好伐？这个主题还支持MaxJax，这解决令我头疼已久的添加公式的大问题！（之前是用MathType+QQ截图 #滑稽）

  • 这个可爱的blog的诞生历程总结，咕咕咕

    • 从零到部署

    1. 下载并安装node.js和git

      在控制台用git -v node -v npm -v 查看是否安装成功

    2. 安装cnpm

      由于npm国内下载速度经常抽风，所以我额外安装淘宝的这个镜像；使用方法就是在命令中把npm换成cnpm即可。

      安装方法:cmd输入命令 npm install -g cnpm --registry=https://registry.npm.taobao.org

    3. 安装Hexo

      cmd 命令cnpm install hexo-cil -g 。安装完后输入hexo -v出现版本信息则安装成功

      创建一个空文件夹（你的博客源码位置）然后Git bash here，输入以下命令

      hexo init

      cnpm install #安装依赖包

      cnpm install hexo-server --save #搭建本地服务器所需插件

      cnpm install hexo-deployer-git --save #使用git方式进行部署博客所需插件

    4. 生成并预览博客

      Git bash here，然后输入以下三连

      hexo clean

      hexo g

      hexo s

      解释一下hexo clean 会清除缓存文件db.json及之前生成的静态文件夹public；

      hexo g会重新生成静态文件夹public。这里的public里面的内容就是到时提交到Github上的静态网页源文件啦，BTW，提交的命令是hexo d。

      最后hexo s就是开启本地预览模式，可以用来看看效果

      打开浏览器，访问http://localhost:4000/，就可以预览啦

    5. 创建GitHub Pages并配置ssh

      1. 创建博客仓库。

        注册并登陆到GitHub官网，假如你注册的用户名是cckk4467，那么新建一个名为cckk4467.github.io的仓库，注意仓库的用户名一定要是用户名+github.io的形式！

      2. 本地生成ssh密钥。
        git bash下输入ssh-keygen -t rsa -C ‘你的邮箱地址’

      3. 上传本地的公钥串，使当前电脑与GitHub账户建立联系。
        在你的电脑C：\ Users\你的计算机用户名.ssh目录下打开刚刚生成的id_rsa.pub，复制里面的内容。然后点击你GitHub账户右上角的头像，选择settings，找到SSH and GPG keys，点击进入之后再点击New SSH key，title随便写，把公钥串粘贴到文本框，保存即可。

    6. 建立本地博客与remote repository联系（这里用Github噢）
      打开你博客根目录下的_config.yml文件，记住这个_config.yml。最下面有一行

      我们就在这里改，添加，改成

    7. 部署到服务器
      继续在根目录Git bash here，输入以下三连
      hexo clean
      hexo g
      hexo d
      注意：第三个命令是hexo d 即将public文件夹中的内容提交到Github上。而不是 hexo s
      此时，访问https://cckk4467.github.io 就可以看到十分简陋的博客啦

    • 更换主题(这里仅以我使用的icarus主题进行展开)

      1. 挑选主题

        进入官网的主题站尽情挑选主题后宫吧。

      2. 进入主题对应的github源码仓库，克隆到博客文件夹themes/themename下

        老位置Git bash here 然后git clone https://github.com/ppoffice/hexo-theme-icarus.git themes/icarus

      3. 使用主题

        打开博客根目录下的_config.yml找到theme: （大概是倒数第二个属性），后面加上主题名字

      4. 配置主题

      5. 重新预览或提交
        hexo clean
        hexo g
        hexo s或者 hexo d

  • 关于绑定自定义域名（我用的是阿里云）

    1. 关于解析域名

      具体操作我就不说了，因注册商而异。关于IPV4的获取，直接ping cckk4467.github.io -4不就行了嘛。
      被github发律师函邮件警告了，所以应该添加CNAME记录，指向cckk4467.github.io这个域名

    2. 关于博客方面的设置

      在博客根目录的source文件夹下，创建一个名为CNAME的文件（无后缀），内容为注册的域名

      完成啦，此时就可以通过自己的域名访问了。
